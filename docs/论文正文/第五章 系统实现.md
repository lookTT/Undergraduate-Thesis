# 第五章 系统实现

本章结合前文的系统总体设计，阐述社区志愿者服务管理系统核心模块在工程环境中的具体实现过程。系统采用前后端分离架构，后端基于 Spring Boot 3 和 Java 21 构建，前端采用 Vue 3、Vite、TypeScript 和 Element Plus 实现。系统开发阶段严格围绕登录认证、志愿者管理、志愿活动管理、报名审核与签到、服务时长统计五个核心功能展开，通过收敛业务边界，确保系统具备可靠的高可用性与完整的功能闭环。

## 5.1 项目初始化与基础环境搭建

系统后端项目位于 [`backend/`](../../backend/) 目录下，采用 Maven 进行依赖管理，核心技术栈包括 Spring Boot 3、Spring Security、JWT、MyBatis-Plus 和 MySQL。后端基础分层按照控制层、服务层、数据访问层和实体层组织，便于后续按业务模块扩展。前端项目位于 [`frontend/`](../../frontend/) 目录下，采用 Vue 3 + Vite 作为工程化方案，并通过 Pinia、Vue Router 和 Axios 完成状态管理、路由控制与请求封装。

为了保证开发环境可复现，系统在文档中固定了 Java 21 与 Node 22 的环境约束。后端数据表通过 `schema.sql` 和 `data.sql` 初始化，启动脚本见 [`docs/script/start-mysql.sh`](../script/start-mysql.sh)，从而保证本地开发环境、数据库结构和初始数据一致。

## 5.2 登录认证功能实现

登录认证是系统的入口功能，也是权限控制的基础。后端在 [`AuthController`](../../backend/src/main/java/com/community/volunteer/controller/auth/AuthController.java) 中提供了 `/api/auth/login`、`/api/auth/logout` 和 `/api/auth/me` 接口。用户提交用户名和密码后，`AuthServiceImpl` 先根据用户名查询用户记录，再使用密码编码器校验密码是否正确；若校验通过，则读取该用户关联的角色信息，并由 JWT 服务生成访问令牌。

JWT 令牌中携带 `userId`、`realName` 和 `role` 等声明，前端在 [`frontend/src/api/auth.ts`](../../frontend/src/api/auth.ts) 和 [`frontend/src/stores/auth.ts`](../../frontend/src/stores/auth.ts) 中完成会话保存与恢复。前端请求封装位于 [`frontend/src/api/http.ts`](../../frontend/src/api/http.ts)，会在请求头中自动附加 `Authorization: Bearer <token>`，从而使后续接口调用能够通过后端鉴权过滤器识别当前登录用户。

登录页实现位于 [`frontend/src/views/LoginView.vue`](../../frontend/src/views/LoginView.vue)。该页面采用双栏布局，左侧展示系统名称和业务说明，右侧提供用户名和密码输入框。用户点击登录后，页面调用后端登录接口，成功后保存 token 和用户信息，并跳转至系统首页。

## 5.3 志愿者管理功能实现

志愿者管理用于维护社区志愿者的基础档案，是后续活动报名、签到和服务统计的基础数据来源。后端在 [`VolunteerController`](../../backend/src/main/java/com/community/volunteer/controller/volunteer/VolunteerController.java) 中提供志愿者列表、详情、新增、修改和删除接口。控制层使用 `@PreAuthorize("hasRole('ADMIN')")` 限制该模块仅管理员可访问，避免普通志愿者越权操作。

业务实现主要集中在 [`VolunteerProfileServiceImpl`](../../backend/src/main/java/com/community/volunteer/service/impl/VolunteerProfileServiceImpl.java) 中。分页查询时，系统从志愿者档案表读取数据并转换为 `VolunteerVO` 返回给前端；新增和修改时，对输入参数进行基础校验后写入数据库。前端列表页位于 [`frontend/src/views/VolunteerView.vue`](../../frontend/src/views/VolunteerView.vue)，采用表格展示志愿者的学号或工号、性别、年龄、所属社区、技能标签和备注等信息，并通过弹窗表单完成新增与编辑操作。

基于 MVP 原则，该模块在首期实现中着重打磨核心链路，仅保留支持志愿服务高频场景的核心体验字段，暂未引入多组织层级嵌套与穿透式审核等复杂形态。这种化繁为简的工程级设计策略，不仅大幅降低了社区初级管理人员的系统使用门槛，还有效保证了核心基础业务的交付质量。

## 5.4 志愿活动管理功能实现

志愿活动管理是系统的核心业务之一，负责活动的创建、发布、修改、删除和查询。后端控制器为 [`ActivityController`](../../backend/src/main/java/com/community/volunteer/controller/activity/ActivityController.java)，其中活动列表和详情允许登录后访问，而活动新增、修改、删除和发布仅管理员可操作。活动创建时，后端从当前登录态中解析出创建者 ID，并写入活动记录，保证活动数据可追溯。

活动实体对应前端页面 [`frontend/src/views/ActivityView.vue`](../../frontend/src/views/ActivityView.vue)。页面顶部提供“新建活动”按钮，弹窗表单支持填写活动标题、活动内容、活动地点、开始时间、结束时间和招募人数。列表区展示活动状态，并为管理员提供编辑、发布和删除操作。

活动状态采用简化枚举方式处理：草稿、已发布和已结束。这样既满足系统展示需要，也便于后续在论文中说明活动流转过程。前端通过 `ActivitySaveRequest` 与后端接口保持一致，减少联调时字段不匹配的风险。

## 5.5 报名审核与签到功能实现

报名审核与签到功能用于完成志愿服务的业务闭环。志愿者在活动发布后可以发起报名请求，管理员则对报名结果进行审核；审核通过后，志愿者在活动现场完成签到，系统自动生成服务记录并累计时长。

后端在 [`ActivitySignupController`](../../backend/src/main/java/com/community/volunteer/controller/activity/ActivitySignupController.java) 中实现报名、报名列表、审核通过和审核拒绝接口，在 [`ActivityCheckinController`](../../backend/src/main/java/com/community/volunteer/controller/activity/ActivityCheckinController.java) 中实现签到和签到列表接口。报名和签到均通过当前登录用户反查志愿者档案，再结合活动状态、审核状态和重复提交判断完成业务校验。

在服务层中，报名逻辑会先检查活动是否已发布，再判断该志愿者是否已报名；签到逻辑会先判断报名是否已审核通过，再判断是否已签到，最后写入签到记录和服务记录。为了避免重复记账，系统对同一活动、同一志愿者的报名和签到都做了唯一性控制。

前端将报名、审核与签到相关操作集中在活动管理相关页面中。普通志愿者在已发布活动列表中可以提交“报名”，并在审核通过后执行“签到”；管理员可以在活动详情的报名列表中完成审核，同时查看报名与签到情况。通过将相关操作集中展示，可以减少不必要的页面跳转，便于用户完成完整流程。

## 5.6 服务时长统计功能实现

服务时长统计用于展示志愿服务成果，并为后续毕业论文的系统测试和效果分析提供数据支撑。系统在志愿者签到成功后自动生成服务记录，记录中包含活动 ID、志愿者 ID、服务时长和记录来源等信息。服务时长由活动开始时间与结束时间计算得到，并在签到时同步写入数据库。

后端在 [`StatisticsController`](../../backend/src/main/java/com/community/volunteer/controller/statistics/StatisticsController.java) 中提供统计概览接口，在 [`ServiceRecordController`](../../backend/src/main/java/com/community/volunteer/controller/statistics/ServiceRecordController.java) 中提供服务记录分页查询接口。管理员登录后，可在仪表盘查看志愿者数量、活动数量、报名数量和累计服务时长等指标；这些指标由服务层从数据库实时汇总得到。

前端仪表盘位于 [`frontend/src/views/DashboardView.vue`](../../frontend/src/views/DashboardView.vue)，通过调用后端统计概览接口获取数据，并以卡片形式展示志愿者数量、活动数量、报名数量和累计服务时长等指标，便于管理员快速了解系统运行概况。

## 5.7 本章小结

本章结合代码结构说明了系统核心模块的实现思路，包括登录认证、志愿者管理、志愿活动管理、报名审核与签到、服务时长统计等。实现过程中遵循前后端分离与后端分层结构，保证接口、数据结构与页面功能保持一致，为第六章的系统测试提供实现基础。
