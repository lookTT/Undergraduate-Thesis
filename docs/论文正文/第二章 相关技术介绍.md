# 第二章 相关技术介绍

本章主要介绍社区志愿者服务管理系统所采用的关键技术，包括后端开发框架、前端开发框架、数据库技术以及认证与权限控制机制。本文选择的技术栈以成熟、稳定、易于实现为原则，能够支撑管理型 Web 系统的快速构建，并满足企业级MVP项目对落地完整性的要求。

## 2.1 Spring Boot

Spring Boot 是当前 Java Web 开发中广泛使用的应用开发框架，其核心特点是约定优于配置，能够帮助开发者快速搭建独立运行的应用程序。与传统 Spring 项目相比，Spring Boot 在依赖管理、自动配置和内嵌服务器方面更加简洁，适合敏捷型开发周期下的中小型管理系统。

本文后端采用 Spring Boot 3.5.4 作为基础框架，结合 Web、Validation、Security、Data Redis 等常用 starter 组件，完成接口开发、参数校验、身份认证和缓存扩展 [1][15]。通过 Spring Boot 统一管理项目结构，可以将控制层、服务层、数据访问层和实体层清晰分离，便于后续按模块化开发与微服务架构拆分。

在本系统中，Spring Boot 主要承担以下职责：

1. 提供 REST 风格接口，供前端页面调用，并按照 OpenAPI 的描述思路组织接口文档 [12]。
2. 集成 Spring Security 完成登录认证与权限控制 [2][13]。
3. 与 MyBatis-Plus 和 MySQL 进行数据交互 [5][11]。
4. 统一处理全局异常、请求参数校验和接口返回格式。

Spring Boot 的使用降低了项目初始化和基础设施搭建的难度，使研发团队可以将更多精力聚焦于核心业务功能和底层逻辑引擎的攻坚上。

## 2.2 Java 21

Java 是本系统后端开发所使用的核心编程语言。本文采用 Java 21 作为开发语言版本，原因在于其属于长期支持版本，兼顾了语言特性、性能稳定性和工具链兼容性 [6][7]。Java 21 与 Spring Boot 3 的适配较好，能够为项目提供更稳定的运行环境。

在系统实现中，Java 21 主要用于编写控制器、服务类、实体类、数据传输对象和工具类。与较早版本相比，Java 21 在语言表达和标准库方面更成熟，能够配合记录类（record）、流式处理等特性提升代码简洁度。本文在登录返回对象、分页对象和统计对象中大量使用 record，以减少样板代码。

Java 21 的优势主要体现在以下几个方面：

1. 语言生态成熟，适合企业级 Web 开发。
2. 与 Spring Boot 3、MyBatis-Plus 等框架兼容良好。
3. 可读性较高，便于研发团队结合源码进行快速联调、压测与代码层面的互相 Review。
4. 版本长期支持，便于本地开发和后续运行环境统一。

因此，Java 21 既满足高兵发系统业务开发需要，也是业界主流云端微服务的稳定技术底座。

## 2.3 Vue 3 与 Vite

Vue 3 是当前流行的前端框架，适合构建交互式页面和后台管理界面。相比 Vue 2，Vue 3 在组合式 API、类型支持和性能方面有明显提升，更适合与 TypeScript 协同开发。Vite 则是新一代前端构建工具，具备启动速度快、热更新快、配置简洁等特点，适合前后端分离项目的快速开发。

本文前端采用 Vue 3.5.13、Vite 7.1.4 和 TypeScript 进行开发，并结合 Vue Router、Pinia、Axios 和 Element Plus 完成页面组织、状态管理、请求封装和组件构建 [3][4][9][10][14]。前端结构主要包括登录页、后台布局页、仪表盘、志愿者管理页和活动管理页，页面之间通过路由切换完成导航。

Vue 3 与 Vite 在本系统中的作用主要包括：

1. 构建组件化页面，提高代码复用率。
2. 通过路由守卫实现登录状态拦截。
3. 通过 Pinia 管理 token 和当前用户信息。
4. 通过 Axios 与后端 REST 接口进行数据交互。
5. 通过 Element Plus 快速实现表格、表单、弹窗和布局。

在管理类系统中，Vue 3 + Vite 方案具有极高的开发与构建编译效率和完美的组件生态协同性，适合作为此类重交互型业务前端的首选技术方案 [3][4]。

## 2.4 MySQL

MySQL 是一种成熟的关系型数据库管理系统，适用于存储结构化业务数据。社区志愿者服务管理系统中的用户、角色、志愿者档案、活动信息、报名记录、签到记录和服务记录等数据都具有较强的结构化特征，因此采用 MySQL 作为数据库是合理的选择。

本文使用 MySQL 8.4 作为数据库版本，并通过初始化脚本创建核心数据表 [5]。数据库表设计遵循业务闭环原则，保证每张表都与系统功能一一对应，避免引入不必要的复杂结构。数据库中的核心表包括用户表、角色表、用户角色关联表、志愿者档案表、活动表、活动报名表、活动签到表和服务记录表。

MySQL 在本系统中的主要作用包括：

1. 持久化保存业务数据。
2. 支撑分页查询与统计汇总。
3. 配合外键逻辑和业务约束保证数据一致性。
4. 为系统底层设计提供明确且可强校验的表结构约束支撑。

通过关系型数据库持久化管控核心业务数据，系统能够更为高效稳妥地支撑复杂表关联的数据检视、宏观统计和可视化BI报表展示，并且在面对后续数据膨胀时可平滑过渡到微服务分布式数据集群方案。

## 2.5 JWT 与权限控制

JWT（JSON Web Token）是一种轻量级的身份认证方案，适合前后端分离架构中的无状态认证场景 [8]。用户登录成功后，服务端生成包含用户身份信息的 token，前端在后续请求中携带该 token，服务端通过解析 token 完成身份识别与权限判断。

本文在认证模块中采用 JWT 与 Spring Security 组合实现基本鉴权 [2][8][13]。登录成功后，后端返回 token、用户 ID、用户名、真实姓名和角色信息；前端将 token 保存在本地，并通过请求拦截器自动附加到请求头中。后端通过 JWT 过滤器提取当前用户身份，并在方法级权限注解 `@PreAuthorize` 的配合下控制接口访问范围。

JWT 与权限控制在本系统中的具体作用如下：

1. 识别当前登录用户。
2. 判断用户角色是否允许访问某个接口。
3. 将管理员和普通志愿者的操作范围区分开来。
4. 降低服务端会话管理复杂度，适合前后端分离系统。

在本系统中，管理员可以执行志愿者管理、活动管理、报名审核、统计查看等操作，而志愿者只能执行活动报名与签到等操作。这样的权限设计既满足实际业务要求，也保证了系统具有基本的安全边界。

## 2.6 本章小结

本章对系统采用的主要技术进行了说明。Spring Boot 提供后端应用基础，Java 21 提供稳定的开发语言支持，Vue 3 与 Vite 提供前端页面与工程化能力，MySQL 用于存储结构化业务数据，JWT 与 Spring Security 则完成认证与权限控制。上述技术共同构成了本文系统实现的基础，也为后续的需求分析、系统设计和系统实现提供了技术依据。
