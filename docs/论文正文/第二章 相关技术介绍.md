# 第二章 相关技术介绍

本章主要介绍社区志愿者服务管理系统所采用的关键技术，包括后端开发框架、前端开发框架、数据库技术、数据访问框架、身份认证机制以及前后端分离架构。本文选择的技术栈以成熟、稳定、易于实现为原则，能够支撑管理型 Web 系统的快速构建，并满足毕业设计项目对落地完整性的要求。各技术在实际系统中的协作关系如图 2.1 所示。

## 2.1 Spring Boot

Spring Boot 是当前 Java Web 开发中广泛使用的应用开发框架，其核心设计哲学是"约定优于配置"（Convention over Configuration），通过将大量繁琐的 XML 配置替换为自动化配置，帮助开发者快速搭建可独立运行的 Spring 应用程序。与传统 Spring Framework 项目相比，Spring Boot 内置了 Tomcat 等嵌入式服务器，无需额外部署容器，依赖管理也通过 parent POM 和 starter 机制统一协调，显著降低了项目初始化成本 [1]。

本文后端采用 Spring Boot 3.x 作为基础框架，结合 spring-boot-starter-web、spring-boot-starter-validation、spring-boot-starter-security 等常用 starter 组件，完成接口开发、参数校验和身份认证 [1][15]。Spring Boot 3 要求 Java 17 及以上版本，本文选用 Java 21 与之配合，能够获得更好的虚拟线程和语言特性支持。通过 Spring Boot 统一管理项目结构，可以将控制层（Controller）、服务层（Service）、数据访问层（Mapper）和实体层（Entity）清晰分离，形成分层架构，便于分模块开发与维护。

在本系统中，Spring Boot 主要承担以下职责：

1. 提供 REST 风格的 HTTP 接口，供前端页面通过 Axios 调用，并按照 OpenAPI 规范的描述思路组织接口结构 [12]。
2. 集成 Spring Security 完成登录认证与基于角色的访问控制 [2][13]。
3. 与 MyBatis-Plus 和 MySQL 进行数据交互，包括 CRUD 操作和分页查询 [5][11]。
4. 通过全局异常处理器（`@RestControllerAdvice`）统一处理异常，通过统一返回对象（Result）规范接口输出格式。

Spring Boot 与其周边生态的整合能力是其广泛应用于管理类系统的核心原因之一。在本系统的开发过程中，Spring Boot 的自动配置机制大幅减少了基础设施配置的工作量，使研发精力得以集中于核心业务功能的实现上。

## 2.2 Java 21

Java 是本系统后端开发所使用的核心编程语言。本文采用 Java 21 作为开发语言版本，原因在于其属于 Oracle 官方指定的长期支持（LTS）版本，兼顾了语言特性、性能稳定性和工具链兼容性 [6][7]。与 Java 17 相比，Java 21 引入了虚拟线程（Virtual Threads，Project Loom）的正式支持，能够以较低的资源开销处理更多并发请求，有利于提升 I/O 密集型服务的吞吐量。

在系统实现中，Java 21 主要用于编写控制器（Controller）、服务类（Service）、实体类（Entity）、数据传输对象（DTO）和工具类。与较早版本相比，Java 21 在语言表达和标准库方面更为成熟，支持密封类（Sealed Classes）、模式匹配（Pattern Matching）和记录类（Record）等现代语言特性，有助于提升代码简洁度与可读性。本文在登录响应对象、分页返回对象和统计汇总对象中使用 record 替代传统的 POJO 类，减少了大量样板代码（Boilerplate Code），使代码结构更加清晰。

Java 21 的主要优势体现在以下几个方面：

1. 语言生态成熟，拥有丰富的第三方库和开发社区支持，适合企业级 Web 开发。
2. 与 Spring Boot 3、MyBatis-Plus 等主流框架兼容良好，无版本冲突风险。
3. record、模式匹配等特性提高了代码可读性，便于后续维护和代码审查。
4. 作为 LTS 版本，版本生命周期长，适合作为长期维护项目的语言底座。

综上，Java 21 能够在满足业务开发需求的前提下，兼顾语言现代性与运行稳定性，是本系统后端开发的合理语言选型。

## 2.3 Vue 3 与 Vite

Vue 3 是 Evan You 主导开发的渐进式前端框架，基于组件化思想构建用户界面，适合快速开发交互式后台管理页面。相比 Vue 2，Vue 3 在核心架构上进行了重写，引入了基于 Proxy 的响应式系统、组合式 API（Composition API）以及对 TypeScript 的一等公民支持，在性能、可维护性和类型安全方面均有显著提升 [3]。

Vite 是 Vue 团队推出的新一代前端构建工具，开发模式下通过原生 ES Module 按需加载，无需打包即可启动开发服务器，热模块替换（HMR）响应速度极快；生产模式下使用 Rollup 进行最终打包，构建效率优于传统 Webpack 方案 [4][14]。

本文前端采用 Vue 3 与 Vite 进行开发，并结合以下配套库构建完整前端工程：

- **Vue Router**：实现前端路由，并通过路由守卫（Navigation Guard）拦截未登录用户，进行登录状态的前端校验。
- **Pinia**：作为 Vue 3 推荐的全局状态管理库，用于持久化保存 token 和当前用户信息，支持 Setup Store 语法与 TypeScript 类型推断。
- **Axios**：基于 Promise 的 HTTP 客户端，通过请求拦截器自动将 token 附加到请求头，通过响应拦截器统一处理错误码和未授权跳转 [10]。
- **Element Plus**：适配 Vue 3 的组件库，提供表格、表单、对话框、分页器、按钮等常用 UI 组件，用于快速搭建管理界面 [9]。

Vue 3 与 Vite 在本系统中的作用主要包括：

1. 构建组件化页面，提高代码复用率，降低页面维护成本。
2. 通过路由守卫实现前端权限拦截，防止未登录用户访问受保护页面。
3. 通过 Pinia 统一管理应用状态，实现登录信息的跨组件共享。
4. 通过 Axios 封装与后端 REST 接口的数据交互，包括统一的错误处理和 token 注入。
5. 通过 Element Plus 快速实现表格、表单、弹窗和布局，降低 UI 开发成本。

## 2.4 MySQL

MySQL 是一种广泛使用的开源关系型数据库管理系统（RDBMS），以其稳定性、高效的查询性能和丰富的生态支持著称，适合承载中小型 Web 系统的结构化业务数据 [5]。关系型数据库的核心优势在于通过表结构、主外键约束和事务机制保证数据的一致性与完整性，适合业务逻辑明确、数据关系较为固定的应用场景。

社区志愿者服务管理系统中的用户、角色、志愿者档案、活动信息、报名记录、签到记录和服务记录等数据具有较强的结构化特征，各实体之间存在明确的关联关系（如一名志愿者可以报名多个活动、一个活动可以有多条报名记录），因此采用 MySQL 作为数据库是合理的选择。

本文使用 MySQL 8.4 LTS 作为数据库版本，并通过初始化 SQL 脚本创建核心数据表 [5]。数据库中的核心表包括用户表（`user`）、志愿者档案表（`volunteer`）、活动表（`activity`）、活动报名表（`enrollment`）、活动签到表（`checkin`）和服务记录表（`service_record`）。数据库表设计遵循业务闭环原则，每张表对应系统中的一个核心实体，表结构以业务最小可用为原则，避免引入不必要的冗余字段。

MySQL 在本系统中的主要作用包括：

1. 持久化保存全部业务数据，保证数据在服务重启后不丢失。
2. 支撑分页查询（`LIMIT/OFFSET`）与统计汇总（`GROUP BY`、`SUM`、`COUNT`）等典型业务查询。
3. 配合 MyBatis-Plus 的代码生成与 ORM 映射，降低 SQL 编写成本。
4. 通过合理的索引设计，保证核心查询在数据规模增长后仍具备良好的响应速度。

## 2.5 MyBatis-Plus

MyBatis-Plus 是基于 MyBatis 的增强型持久层框架，在不改变 MyBatis 原有功能的前提下，通过提供通用 Mapper、条件构造器（Wrapper）、分页插件、代码生成器等功能，显著简化了数据访问层的开发工作 [11]。与纯 MyBatis 相比，MyBatis-Plus 内置了常用的 CRUD 方法，开发者无需重复编写简单的单表操作 SQL，同时仍然可以在 Mapper.xml 或注解中自定义复杂查询，兼顾了开发效率与灵活性。

在本系统中，MyBatis-Plus 的使用方式如下：

1. 所有 Mapper 接口继承 `BaseMapper<T>`，自动获得 `insert`、`deleteById`、`updateById`、`selectById`、`selectList` 等通用方法，无需额外编写 XML。
2. 使用 `LambdaQueryWrapper` 构建类型安全的查询条件，避免字符串拼接带来的字段名错误风险。
3. 引入分页插件（`MybatisPlusInterceptor`），在不修改业务代码的情况下实现统一分页，前端只需传入页码和每页数量即可获得分页结果。
4. 活动报名、签到查询等涉及多表关联的复杂场景，通过自定义 SQL 语句在 Mapper.xml 中实现，与 MyBatis-Plus 的通用方法共存，互不干扰。

MyBatis-Plus 的引入使得系统中单表查询和基础 CRUD 操作的代码量大幅减少，开发效率明显提升，同时保留了对复杂业务查询的自定义能力，适合本系统的开发目标。

## 2.6 JWT 与权限控制

JWT（JSON Web Token）是一种基于 JSON 的开放标准（RFC 7519），用于在各方之间安全地传递声明信息 [8]。JWT 由头部（Header）、载荷（Payload）和签名（Signature）三部分构成，通过 Base64URL 编码拼接而成，整个 token 是一个可验证但不可伪造的字符串。在前后端分离架构中，JWT 通常作为无状态身份认证的载体：用户完成登录后，服务端签发包含用户 ID 和角色信息的 token，客户端将其存储在本地并在后续请求的 Authorization 头中携带，服务端只需验签即可识别当前用户身份，无需服务端维护 Session，适合横向扩展的分布式系统。

本文在认证模块中采用 JWT 与 Spring Security 组合实现基本鉴权 [2][8][13]。整体认证流程如下：用户提交用户名和密码，后端通过 Spring Security 的 AuthenticationManager 完成身份验证；验证通过后，使用 JJWT 库生成 token 并返回给前端；后续请求经过自定义的 JwtAuthenticationFilter，该过滤器解析 token、提取用户信息并将 SecurityContext 填充，Spring Security 的方法级权限注解 `@PreAuthorize` 即可基于此上下文完成接口级权限控制。

本系统的权限模型采用基于角色的访问控制（RBAC，Role-Based Access Control）[16]。系统中设置管理员（ADMIN）和志愿者（VOLUNTEER）两种角色，不同角色对应不同的接口权限：

- 管理员：拥有志愿者管理、活动管理、报名审核、签到管理和统计查看的全部接口访问权限。
- 志愿者：仅能访问活动列表查看、活动报名和签到提交等与个人参与相关的接口。

这种基于角色的权限控制设计既满足了实际业务的安全隔离需求，也降低了权限模型的复杂度，便于后续在角色数量增加时平滑扩展 [16]。

## 2.7 前后端分离架构

前后端分离是当前 Web 应用开发的主流架构模式之一。在该架构下，前端与后端通过 HTTP/REST 接口进行交互，各自独立开发、独立部署，解耦了界面渲染与业务逻辑。与传统服务端渲染（SSR）方式相比，前后端分离架构具有开发并行效率高、前端技术栈灵活可替换、后端接口可同时服务多种客户端（Web、移动端等）等优势，适合对交互体验有一定要求的管理类系统。

本系统后端基于 Spring Boot 提供 RESTful API，前端通过 Vue 3 + Axios 消费接口。两端之间的通信遵循统一的接口规范：请求使用标准 HTTP 方法（GET/POST/PUT/DELETE），响应采用统一的 JSON 结构（包含状态码、提示信息和数据体），身份认证通过 Authorization 请求头中的 JWT token 完成。这一架构模式使前后端团队可以并行推进开发，以 Swagger/OpenAPI 文档作为接口契约对齐联调细节，有效降低了沟通成本和集成风险 [12]。

前后端分离架构的主要优势在本系统中体现为：

1. 后端可专注于业务逻辑、数据持久化和安全控制，前端可专注于用户界面和交互体验，职责明确。
2. 前端构建产物可通过 Nginx 等静态服务器独立托管，后端服务可按需横向扩展，部署灵活。
3. 接口文档通过 Swagger 自动生成，降低了手工维护接口文档的成本，也便于后续接口的验证与测试。

## 2.8 本章小结

本章对系统采用的主要技术进行了详细介绍。Spring Boot 提供后端应用框架基础，Java 21 提供稳定的核心开发语言支持，Vue 3 与 Vite 提供前端页面构建与工程化能力，MySQL 用于存储结构化业务数据，MyBatis-Plus 简化了数据访问层的开发，JWT 与 Spring Security 完成身份认证与基于角色的权限控制，前后端分离架构则明确了系统的整体组织模式。上述技术相互配合，共同构成了本系统实现的技术基础，也为后续的需求分析、系统设计和系统实现提供了可靠的技术依据。
