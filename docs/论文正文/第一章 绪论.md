# 第一章 绪论

## 1.1 研究背景

随着社区治理体系和基层服务能力不断完善，志愿服务在社区公共事务、便民活动、文明宣传和应急协助等场景中的作用越来越明显。志愿者数量逐步增加后，传统的纸质登记、微信群通知、人工统计等管理方式已经难以满足日常管理需要，容易出现信息分散、报名重复、签到统计不准确、服务时长统计滞后等问题。

在实际社区志愿服务管理过程中，活动发布、志愿者报名、审核与签到往往由不同人员分别处理。如果缺少统一的信息化平台，活动状态、报名名单、签到记录和服务时长等数据就会分散在多个表格或记录中，不仅增加管理人员工作量，也不利于后续的数据汇总与成果展示。因此，建设一个面向社区场景的志愿者服务管理系统具有较强的现实需求。

从技术发展角度看，Java Web、前后端分离架构、关系型数据库和权限控制框架已经较为成熟，可以支持快速构建稳定的业务系统。Spring Boot 适合快速搭建后端服务，Vue 3 与 Vite 适合实现响应式的前端页面，MySQL 适合承载结构化业务数据，JWT 和 Spring Security 则可以实现较为清晰的身份认证与访问控制。基于这些成熟技术构建社区志愿者服务管理系统，既具备实现可行性，也便于后续在论文中完整说明系统设计与实现过程。

## 1.2 研究意义

社区志愿者服务管理系统的建设具有一定的实际应用价值。首先，系统能够将志愿者档案、活动信息、报名审核、签到和服务记录统一管理，减少人工统计成本，提高管理效率。其次，系统可以通过权限控制将管理员和志愿者的操作范围区分开来，避免越权操作和重复录入问题，提高数据可靠性。再次，系统通过统计模块对志愿服务成果进行集中展示，便于社区管理人员了解活动开展情况和志愿服务积累情况。

从毕业设计角度看，该题目适合作为“设计与实现”类论文课题。系统功能范围清晰，既包含登录认证、CRUD 管理、状态流转、统计汇总等典型业务，又不会因为范围过大而导致实现失控。通过该课题，能够较完整地体现需求分析、系统设计、数据库设计、接口设计、权限控制、系统实现和系统测试等软件工程环节，符合本科毕业设计的训练目标。

此外，该系统还具有一定的示范意义。社区志愿者服务管理系统可以作为基层信息化管理场景的一个缩影，论文中所采用的前后端分离架构、角色权限控制和数据统计思路，也可以迁移到其他类型的中小型管理系统中。

## 1.3 国内外研究现状

从现有应用情况来看，国内外与志愿服务管理相关的信息系统大多围绕志愿者注册、活动发布、活动报名、任务分配、签到确认和服务时长统计等功能展开。国外相关平台通常更强调志愿活动组织、社会资源协同和跨机构参与，系统设计上更注重可扩展性和开放接口；国内相关系统则更多服务于校园、社区和公益组织等具体场景，功能上更强调日常管理、数据汇总和审批流程。

在技术实现方面，志愿服务管理系统已经逐渐从早期的单机程序或传统 Web 页面，发展为前后端分离的业务系统。后端通常采用 Spring Boot、Django、Node.js 等框架提供 REST 接口，前端则使用 Vue、React 等技术构建交互界面，数据库多采用 MySQL 或 PostgreSQL 存储结构化数据。这类技术路线的优势在于开发效率高、接口清晰、便于维护，也有利于后期扩展统计分析和权限管理能力。

从功能完整度来看，现有系统往往已经覆盖基础登记和活动管理，但针对具体社区或毕业设计场景时，功能粒度常常偏大，或者偏向展示而缺少完整业务闭环。对毕业设计而言，更重要的不是功能数量，而是能否围绕一个明确业务场景，完成从登录、档案管理、活动管理、报名审核、签到到统计的完整链路。因此，本文在吸收常见志愿服务管理系统设计思路的基础上，选择精简版核心功能作为实现目标，以确保系统可落地、论文可写实、测试可验证。

## 1.4 本文主要工作

围绕社区志愿者服务管理这一业务场景，本文完成了以下主要工作：

1. 明确系统的业务范围与角色划分，确定管理员和志愿者两类核心角色。
2. 基于 Spring Boot 3、Java 21、Vue 3 和 MySQL 设计并实现前后端分离系统。
3. 完成登录认证、JWT 鉴权和基于角色的访问控制，保证系统具备基本安全性。
4. 设计并实现志愿者管理、志愿活动管理、报名审核与签到、服务时长统计等核心功能。
5. 构建与系统实现一致的数据库结构和接口设计，保证论文与代码内容一致。
6. 结合实际运行结果完成系统测试，对功能、权限和异常场景进行验证。

本文的实现重点放在最小可用闭环上，即保证系统能够真实运行、真实演示和真实测试，而不是单纯扩展功能数量。这样既符合毕业设计要求，也有利于在有限时间内完成论文撰写与系统开发。

## 1.5 论文结构安排

全文共分为七章，各章内容安排如下：

第一章为绪论，主要介绍课题研究背景、研究意义、国内外研究现状、本文主要工作以及论文结构安排，为全文提供总体引导。

第二章为相关技术介绍，主要对 Spring Boot、Java 21、Vue 3、Vite、MySQL、JWT 与权限控制等技术进行说明，为后续系统设计与实现奠定技术基础。

第三章为系统需求分析，主要分析社区志愿者服务管理系统的业务需求、角色需求、功能需求、非功能需求和可行性，明确系统需要解决的问题。

第四章为系统设计，主要包括系统总体架构设计、功能模块设计、数据库设计、接口设计以及权限与安全设计，为系统实现提供结构依据。

第五章为系统实现，结合实际代码与运行结果，说明登录认证、志愿者管理、活动管理、报名审核与签到、服务时长统计等核心功能的实现过程。

第六章为系统测试，围绕功能测试、权限测试、异常测试等内容，对系统运行效果进行验证，并分析测试结果。

第七章为总结与展望，对全文工作进行归纳，分析系统存在的不足，并提出后续可扩展方向。

## 1.6 本章小结

本章从社区志愿服务管理的现实需求出发，说明了课题提出的背景和研究意义，并结合当前信息化管理趋势分析了志愿服务管理系统的实现方向。与此同时，本文明确了自身的实现范围和论文结构，为后续章节的需求分析、系统设计、系统实现和系统测试奠定了基础。
