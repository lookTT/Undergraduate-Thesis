# 接口清单

## 1. 接口设计原则

- 接口围绕核心业务闭环设计。
- 优先覆盖登录、志愿者、活动、报名、签到、统计。
- 接口命名保持简单、可读、可维护。

## 2. 认证与用户接口

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | `/api/auth/login` | 用户登录 |
| POST | `/api/auth/logout` | 用户退出 |
| GET | `/api/auth/me` | 获取当前登录用户信息 |
| PUT | `/api/auth/password` | 修改密码 |

## 3. 志愿者管理接口

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/api/volunteers` | 查询志愿者列表 |
| GET | `/api/volunteers/{id}` | 查询志愿者详情 |
| POST | `/api/volunteers` | 新增志愿者档案 |
| PUT | `/api/volunteers/{id}` | 修改志愿者档案 |
| DELETE | `/api/volunteers/{id}` | 删除志愿者档案 |

## 4. 志愿活动接口

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/api/activities` | 查询活动列表 |
| GET | `/api/activities/{id}` | 查询活动详情 |
| POST | `/api/activities` | 新增活动 |
| PUT | `/api/activities/{id}` | 修改活动 |
| DELETE | `/api/activities/{id}` | 删除活动 |
| PUT | `/api/activities/{id}/publish` | 发布活动 |

## 5. 报名审核接口

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | `/api/activities/{id}/signup` | 志愿者报名活动 |
| GET | `/api/activities/{id}/signups` | 查询活动报名列表 |
| PUT | `/api/signups/{id}/approve` | 审核通过报名 |
| PUT | `/api/signups/{id}/reject` | 审核拒绝报名 |

## 6. 签到与统计接口

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | `/api/activities/{id}/checkin` | 活动签到 |
| GET | `/api/records` | 查询服务记录列表 |
| GET | `/api/statistics/summary` | 获取统计概览 |
| GET | `/api/statistics/volunteer/{id}` | 获取某志愿者统计信息 |

## 7. 接口返回约定

- 统一返回结构建议包含：`code`、`message`、`data`
- 列表接口建议返回分页信息
- 失败时返回明确错误信息，避免前后端联调困难

## 8. 接口实现优先级

1. 认证接口
2. 志愿者接口
3. 活动接口
4. 报名审核接口
5. 签到与统计接口

## 9. 说明

- 这是当前精简版项目的最小接口集。
- 如果后续仍想进一步压缩工作量，可以将“删除接口”和“查询详情接口”延后实现，只保留增改查和核心业务流转接口。
