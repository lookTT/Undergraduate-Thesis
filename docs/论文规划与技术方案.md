# 论文规划与技术方案

## 1. 题目确认

- 题目：基于 Spring Boot 的社区志愿者服务管理系统的设计与实现
- 目标：完成一个前后端分离的社区志愿者服务管理系统，并形成符合学校毕业论文要求的设计与实现型论文。

## 2. 本机环境确认

- Java：OpenJDK 21.0.10 LTS
- `javac`：21.0.10
- Node：v22.18.0

结论：

- Java 21 可直接用于 `Spring Boot 3`
- Node 22 可直接用于主流前端工程化方案

## 3. 技术栈方案

### 3.1 后端

- `Spring Boot 3`
- `Java 21`
- `Spring Security`
- `JWT`
- `MyBatis-Plus`
- `MySQL`
- `Swagger / OpenAPI`

### 3.2 前端

- `Vue 3`
- `Vite`
- `TypeScript`
- `Pinia`
- `Vue Router`
- `Element Plus`

### 3.3 选型理由

- `Vue 3 + Vite` 对管理后台开发效率高，资料多，适合毕业设计。
- `Element Plus` 适合做后台管理界面，表格、表单、弹窗等基础能力成熟。
- `Spring Boot 3 + Java 21` 与当前本机环境一致，开发和部署成本低。
- `MyBatis-Plus + MySQL` 适合快速完成 CRUD 型业务系统，便于论文中描述数据库设计和接口设计。

## 4. 系统定位

系统类型：

- 社区志愿者服务管理后台

核心目标：

- 管理志愿者信息
- 管理志愿活动发布、报名、审核、签到
- 统计服务时长与活动成果
- 支撑社区日常志愿服务管理闭环

## 5. 功能规划

### 5.1 基础功能

- 用户登录与退出
- 角色权限控制
- 修改密码（预留扩展功能）

### 5.2 志愿者管理

- 志愿者档案管理
- 志愿者信息查询

### 5.3 活动管理

- 志愿活动发布
- 活动报名
- 报名审核
- 活动签到

### 5.4 服务统计

- 服务时长累计
- 简单统计页面

## 6. 数据库设计草案

建议的核心表：

- `user`：用户表
- `role`：角色表
- `user_role`：用户角色关联表
- `volunteer_profile`：志愿者档案表
- `activity`：志愿活动表
- `activity_signup`：活动报名表
- `activity_checkin`：活动签到表
- `service_record`：服务记录表

设计原则：

- 以业务闭环为主，不做过度复杂的表结构
- 能支撑论文中的“需求分析、系统设计、实现、测试”四个部分
- 所有表结构需要服务于实际功能，不做无意义冗余字段

## 9. 学校要求提醒

- 正文篇幅要满足要求，理工类一般不少于 5000 字，按论文标准尽量写到 1 万字以上。
- 参考文献不少于 15 篇。
- 外文文献不少于 3 篇。
- 近五年文献占比要足够高。
- 学位论文的查重和 AIGC 检测要控制在要求范围内。
- 写作内容必须与实际项目一致，避免论文和代码脱节。

## 10. 当前结论

- 后端采用 `Spring Boot 3 + Java 21`
- 前端采用 `Vue 3 + Vite + TypeScript + Element Plus`
- 项目方向为社区志愿者服务管理系统，采用精简版核心功能
- 论文与项目按“需求 - 设计 - 实现 - 测试 - 总结”闭环推进
